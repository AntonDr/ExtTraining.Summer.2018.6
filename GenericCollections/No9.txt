- Разработать обобщенный типизированный класс-коллекцию Set<T> (решение поместить в проект 
GenericCollections), реализующий 
	- методы Add, Remove, Contains (с вычислительной сложностью O(1) или O(log n)) и т.д.;
	- основные операции над множествами (ISet<T>);
	- возможность перечисления элементов коллекции. 
- В качестве внутренней структуры для хранения элементов доускается использование только 
кастомных типов и (или) sz-массивов.
- Предложить альтернативные статические методы для основных операций над множствами, например,
результат объединения двух множеств - новое множество (Union как альтернатива UnionWith (ISet<T>)).
- Протестировать методы разработанного класса, тесты поместить в проект GenericCollections.Tests.
Метод считается методом коллекции только в том случае, когда он покрыт unit-тестами.
- Учитывая большое количесиво членов, которые обязательно должны присутствовать в классе
(с NotImplementedException, если не реализованы), обязательными для реализации и тестирования
являются методы Add, Remove, Contains, UnionWith, Union, остальные - по возможности.
